## Workflow (Intake to Publish)

**Phase 1: Topic Discovery (Agent 1 - 2-5 minutes)**
1.1 Analyze domain focus: data, L&D, gamification, AI
2.2 Search current trends and search volume data
3.3 Identify 3 high-potential topics
4.4 Rank by relevance + ranking opportunity
5.5 Reject off-focus topics

**Phase 2: Keyword and SERP Analysis (Agent 2 - 5-10 minutes)**
1.1 Map keyword clusters and supporting entities
2.2 Determine search intent (informational, commercial, navigational)
3.3 Analyze competitor content coverage gaps
4.4 Identify internal link opportunities
5.5 Define primary keyword with volume estimates

**Phase 3: Brief Development (Agent 3 - 3-5 minutes)**
1.1 Develop unique angle based on gap analysis
2.2 Create 6-section structure with subsections
3.3 Define thesis statement and audience profile
4.4 Plan CTA placement and evidence requirements
5.5 Document content strategy and style requirements

**Phase 4: Research and Evidence Gathering (Agent 4 - 8-12 minutes)**
1.1 Gather 6+ credible sources (academic papers, industry reports, case studies)
2.2 Create evidence-backed claims table
3.3 Document references for each major section
4.4 Extract practical examples and data points
5.5 Validate source credibility and recency

**Phase 5: Draft Writing (Agent 5 - 10-15 minutes)**
1.1 Write 200-300 words per section (total 1200-1800 words)
2.2 Integrate core concepts with 1.9% keyword density
3.3 Use scannable bullets and formatting
4.4 Incorporate internal links naturally
5.5 Include CTA in appropriate section

**Phase 6: SEO Optimization (Agent 6 - 3-5 minutes)**
1.1 Optimize title, meta description, and slug
2.2 Validate heading hierarchy (H1, H2, H3)
3.3 Generate Schema JSON-LD structure
4.4 Integrate internal links
5.5 Add alt text for images if applicable

**Phase 7: Editor Review (Agent 7 - 5-8 minutes)**
1.1 Improve readability score from 65 to 70+ threshold
2.2 Remove all humanizer patterns (em dashes, title-case headings, robotic transitions)
3.3 Enhance clarity and authority
4.4 Resolve any quality issues

**Phase 8: Refresh Planning (Agent 8 - 2-3 minutes)**
1.1 Define quarterly refresh schedule
2.2 Identify post-publication triggers
3.3 Set review cadence

5.5 Document maintenance strategy

**Phase 9: Publishing QA (Agent 9 - 2-3 minutes)**
1.1 Run pre-publish quality checks
2.2 Execute publish script

## Automation Cadence and Backlog Rules

**Weekly Operating Rhythm**:

**Monday: Sprint Planning**
- Review backlog and select next topic
- Assign agent roster for upcoming sprint
- Review team_notes from previous sprints
- Set sprint objectives and success metrics

**Tuesday-Thursday: Sprint Execution**
- Execute one complete sprint per day
- Move agents in strict sequence
- Document findings in team_notes
- Review quality gates between agents

**Friday: Review and Optimize**
- Review completed sprints
- Update team_notes with learnings
- Optimize processes based on results
- Plan next week's priorities

**Monthly: Deep Analysis**
- Review published posts and metrics
- Review ranking performance
- Update research materials
- Refresh outdated content

**Sprint Backlog Rules**:

**Topic Selection Priority**:
1. High search volume + moderate competition (golden opportunity)
2. Gap analysis results (what's missing vs. competitors)
3. Personal expertise and practical examples available
4. Trend alignment with current data/LxD/gamification trends
5. Content evergreen potential (3+ month lifespan)

**Topic Rejection Criteria**:
- Off-topic (not data, LxD, gamification, AI)
- Already extensively covered by competitors
- Requires specialized knowledge not available
- Short content lifecycle (trend-based only)
- No ranking opportunity (extremely high competition)

**Backlog Management**:
- Minimum 3 topics ready for sprint selection
- Each topic pre-vetted for ranking opportunity
- Source materials gathered in advance
- Brief templates available for common content types

**Resource Planning**:
- Allocate 40 minutes per sprint (including handoff pauses)
- Reserve 30 minutes per sprint for publishing QA
- Schedule regular review sessions to refine process
- Build in buffer time for unexpected issues

## Failure Handling and Fallback Logic

**Agent-Level Failure Scenarios**:

**Topic Scout Failure**:
- **Issue**: Cannot identify 3 viable topics in domain
- **Fallback**: Expand domain to include adjacent data topics
- **Recovery**: Request human guidance for topic direction
- **Prevention**: Maintain topic repository with 10+ vetted candidates

**Keyword Analyst Failure**:
- **Issue**: No viable keywords found
- **Fallback**: Select from high-volume generic keywords
- **Recovery**: Re-analyze SERP from different angles
- **Prevention**: Build keyword database with search volume estimates

**Brief Architect Failure**:
- **Issue**: Cannot create unique angle
- **Fallback**: Use standard content template
- **Recovery**: Request creative direction from human
- **Prevention**: Develop multiple brief templates for different content types

**Research Analyst Failure**:
- **Issue**: Cannot find credible sources
- **Fallback**: Use industry reports and authoritative blogs
- **Recovery**: Expand search terms and sources
- **Prevention**: Build curated source library

**Writer Failure**:
- **Issue**: Cannot draft compelling content
- **Fallback**: Request human outline or partial draft
- **Recovery**: Focus on clarity over creativity
- **Prevention**: Provide detailed brief and examples

**SEO Optimizer Failure**:
- **Issue**: Cannot optimize metadata
- **Fallback**: Use template-based optimization
- **Recovery**: Manual SEO review by human
- **Prevention**: Build template library

**Editor Failure**:
- **Issue**: Cannot improve readability
- **Fallback**: Request human review
- **Recovery**: Implement basic readability improvements
- **Prevention**: Use readability scoring tools

**Refresh Planner Failure**:
- **Issue**: Cannot define refresh schedule
- **Fallback**: Use standard quarterly schedule
- **Recovery**: Request human guidance for specific needs
- **Prevention**: Maintain standard schedule templates

**Publishing QA Failure**:
- **Issue**: Publication blocked
- **Fallback**: Fix identified issues and retry
- **Recovery**: Escalate to human for manual publish
- **Prevention**: Thorough pre-publish validation

**Process-Level Failure Scenarios**:

**Performance Degradation** (>5 minutes per agent):
- **Issue**: Agent execution time exceeding targets
- **Fallback**: Streamline agent responsibilities
- **Recovery**: Analyze bottlenecks and optimize
- **Prevention**: Implement time tracking and alerts

**Quality Gates Failure**:
- **Issue**: Cannot pass quality gates
- **Fallback**: Request human intervention for critical issues
- **Recovery**: Address specific failures and retry
- **Prevention**: Implement automated quality checks

**External Dependency Failure**:
- **Issue**: Source material unavailable
- **Fallback**: Use alternative sources
- **Recovery**: Document alternative sources
- **Prevention**: Maintain source alternatives

**Measurement Gap**:
- **Issue**: No baseline metrics available
- **Fallback**: Use industry benchmarks
- **Recovery**: Collect initial metrics post-publication
- **Prevention**: Track metrics for all published content

**Recovery Workflow**:
1. Identify failure mode and root cause
2. Apply appropriate fallback strategy
3. Document failure and recovery in team_notes
4. Implement prevention for future sprints
5. Continue from point of failure or restart sprint if critical
6. Notify human if fallback fails or creates quality issues
