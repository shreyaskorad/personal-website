## Failure Modes and Recovery

**Failure Mode 1: Notebook Not Available**
- **Detection**: Trigger fires, no notebook accessible
- **Recovery**:
  1. Use mental placeholder method: "___" for later
  2. Set specific time to capture later
  3. Put reminder note on device
  4. Check next capture opportunity
  5. Never skip; always capture eventually
- **Prevention**: Multiple capture locations, backup methods, "always have notebook ready" habit

**Failure Mode 2: Entry Quality Declines**
- **Detection**: Entries become too brief or meaningless
- **Recovery**:
  1. During evening processing, expand all entries
  2. Use Method C expansion template
  3. Add context and priority to all entries
  4. Review week's entries for patterns
  5. Restart quick capture with quality focus
- **Prevention**: Expand quickly during evening, use quality checks, never postpone expansion

**Failure Mode 3: Streak Breaks**
- **Detection**: Missed capture for 2+ consecutive days
- **Recovery**:
  1. Acknowledge without guilt
  2. Set new streak goal (usually 3 days minimum)
  3. Simplify system for restart
  4. Use small reward to motivate restart
  5. Learn and adjust to prevent recurrence
- **Prevention**: Multiple capture locations, low-friction system, trigger reminders, positive reinforcement

**Failure Mode 4: Trigger Fatigue**
- **Detection**: Captures feel like chore, motivation declines
- **Recovery**:
  1. Reduce trigger count temporarily
  2. Simplify entry method
  3. Focus on 1-2 triggers that work best
  4. Use rewards and celebrations
  5. Take 1-day break if needed
- **Prevention**: Rotate triggers, vary rewards, keep entry simple, focus on quality over quantity

**Failure Mode 5: Overwhelmed by Volume**
- **Detection**: Too many entries to process, processing becomes impossible
- **Recovery**:
  1. Process in batches (morning/evening)
  2. Archive completed items weekly
  3. Reduce trigger intensity
  4. Focus on high-value entries first
  5. Simplify processing framework
- **Prevention**: Realistic trigger count, processing limits, regular reviews, automated organization

**Failure Mode 6: Entry Ambiguity**
- **Detection**: Entries unclear, no context or meaning later
- **Recovery**:
  1. During evening processing, add context
  2. Expand to Method C format
  3. Add date and time if missing
  4. Add priority level
  5. Tag for follow-up if needed
- **Prevention**: Use Method C for new entries, expand quickly, always add context

**Failure Mode 7: Habit Abandonment**
- **Detection**: Missing practice for 1+ week
- **Recovery**:
  1. Review what went wrong
  2. Restart with simplified system
  3. Focus on 1 trigger initially
  4. Use rewards to motivate restart
  5. Set realistic expectations
- **Prevention**: Keep system simple, realistic triggers, rewards system, celebrate successes
