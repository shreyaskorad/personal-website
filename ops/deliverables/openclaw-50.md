## Prioritization Rules

**Eisenhower Matrix for Project Tasks**:
```
┌─────────────────────┬─────────────────────┬─────────────────────┐
│                    │ IMPORTANT          │ IMPORTANT          │
│                    │                    │                    │
│ NOT IMPORTANT     │ Q2: Schedule       │ Q1: Do First       │
│ Urgent             │                    │                    │
│                    │                    │                    │
├─────────────────────┼─────────────────────┼─────────────────────┤
│                    │ IMPORTANT          │ IMPORTANT          │
│                    │                    │                    │
│ URGENCY            │ Q3: Delegate       │ Q4: Don't Do       │
│ NOT URGENT         │                    │                    │
│                    │                    │                    │
└─────────────────────┴─────────────────────┴─────────────────────┘

Priority Logic:
- Q1 (Do First): Urgent AND Important - Critical path tasks
- Q2 (Schedule): Important but Not Urgent - Strategic projects requiring planning
- Q3 (Delegate): Urgent but Not Important - Can be delegated or automated
- Q4 (Don't Do): Not Urgent AND Not Important - Eliminate or reduce
```

**50/30/20 Rule for Time Allocation**:
- **50%**: High-priority strategic work (prototype development, critical reviews)
- **30%**: Ongoing operational work (maintenance, meetings, communications)
- **20%**: Future-forward activities (planning, learning, networking)

**Daily Priority Hierarchy**:
1. Critical path tasks with strict deadlines
2. Tasks blocking others' progress
3. Important tasks requiring significant focus
4. Administrative tasks and communications
5. Low-priority tasks and nice-to-dos

## Review Cadence

**Daily**: End of each work session
- Update task completion status
- Identify blocking issues
- Plan next day priorities
- Track time allocation

**Weekly**: Friday 4:00 PM
- Review week progress against priorities
- Update project status dashboards
- Identify risks and blockers
- Plan next week's priorities
- Communicate status to stakeholders
- Schedule manager review meeting

**Monthly**: End of each month
- Comprehensive project portfolio review
- Evaluate progress on strategic goals
- Adjust priorities and timelines
- Identify long-term risks
- Update resource allocation
- Plan quarterly priorities

**Quarterly**: End of each quarter
- Strategic review of all projects
- Evaluate business impact
- Adjust project portfolio
- Identify new opportunities
- Plan next quarter strategy
- Update quarterly goals and metrics

**Quarterly Manager Review**:
- Share progress on key projects
- Discuss challenges and blockers
- Align on priorities for next period
- Get strategic guidance
- Validate assumptions and plans
- Adjust timelines and resources
