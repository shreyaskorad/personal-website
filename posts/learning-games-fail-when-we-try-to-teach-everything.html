Learning games fail when we try to teach everything

April 15, 2026 · 2 min read · ai, modern l&d, behaviour science, gamification

If your learning game tries to cover ten models at once, people leave with noise, not capability. Design for one high value behaviour, then make the game rehearse it in context.

In a Professor Game episode, Leif Sorensen described a familiar client request: add teamwork, leadership, ten frameworks, and a pile of content to a single learning game. He called it the magic powder mistake. The result is usually a game that feels impressive, and teaches little.

This is not just a taste issue. Working memory is limited. A learning game already spends cognitive budget on rules, navigation, and feedback. When you stack objectives and models on top, you increase cognitive load and reduce what learners can actually encode and use later.

Extra material also creates a seductive details problem. The additional frameworks look valuable, so people attend to them, but that attention competes with the core skill you wanted. You get recognition without recall, and discussion without behaviour change.

Transfer is the real bar. Baldwin and Ford's classic review shows transfer depends on what you practice, how you practice it, and the conditions learners return to at work. So pick one behaviour, build the game around 3 to 5 decisions, then debrief. If you want help turning a messy objective list into a transfer focused brief, reach out.

Start with one behaviour, make it feel real, and ship a level two later.
