Primary Objective: Execute a complete, serialized SEO blog cycle for the topic "AI framework comparison with decision matrix for leaders" to increase organic traffic through high-intent, topic-relevant content that demonstrates expertise across LxD, gamification, data, and AI domains while maintaining quality standards and measurable SEO performance.

Success Metrics:

Traffic Growth: 25% increase in organic traffic within 90 days of publication

Agent 1: Topic Scout (2-5 minutes) - Role: Strategic topic identification and prioritization for AI, LxD, gamification, data domains - Output: Prioritized topic slate with thesis, audience intent, and ranking opportunity assessment - Success Criteria: 3 candidates presented with ranking rationale, off-focus topics explicitly rejected

Agent 2: Keyword and SERP Analyst (5-10 minutes) - Role: Technical SEO analysis and gap identification - Output: Primary keyword, supporting entities, keyword clusters, SERP gap analysis, internal-link targets - Success Criteria: Primary keyword defined with volume, search intent classification, gaps identified

Agent 3: Brief Architect (3-5 minutes) - Role: Convert research into actionable brief and outline with unique angle - Output: Approved brief with 6-section structure, CTA placement, evidence requirements - Success Criteria: Brief approved with unique angle, 6-section structure, target audience profile

Phase 1: Topic Discovery (Agent 1 - 2-5 minutes) 1.1 Analyze domain focus: AI, LxD, gamification, data 2.2 Search current trends and search volume data 3.3 Identify 3 high-potential topics 4.4 Rank by relevance + ranking opportunity 5.5 Reject off-focus topics

Phase 2: Keyword and SERP Analysis (Agent 2 - 5-10 minutes) 1.1 Map keyword clusters and supporting entities 2.2 Determine search intent (informational, commercial, navigational) 3.3 Analyze competitor content coverage gaps 4.4 Identify internal link opportunities 5.5 Define primary keyword with volume estimates

Phase 3: Brief Development (Agent 3 - 3-5 minutes) 1.1 Develop unique angle based on gap analysis 2.2 Create 6-section structure with subsections 3.3 Define thesis statement and audience profile 4.4 Plan CTA placement and evidence requirements 5.5 Document content strategy and style requirements

Agent-Level Quality Gates:

Topic Scout Gate: - [ ] 3 candidates presented with ranking rationale - [ ] Off-focus topics explicitly rejected - [ ] Topics aligned with domain focus

Keyword Analyst Gate: - [ ] Primary keyword defined with volume range - [ ] Search intent clearly classified - [ ] SERP gap identified - [ ] Internal link targets mapped

Pre-Publish Validation: - [ ] Content aligns with approved brief - [ ] Tone matches site voice (authoritative, practical, conversational) - [ ] Length within 1200-1800 word range - [ ] Keyword density at 1.8-2.0% - [ ] No remaining humanizer patterns - [ ] All sections present and complete - [ ] Keywords positioned naturally - [ ] All evidence-backed claims documented - [ ] CTA included and positioned appropriately - [ ] Internal links integrated and valid - [ ] Alt text added for images - [ ] Meta tags optimized (title 50-60 chars, description 150-160 chars)

Technical Publishing Check: - [ ] Publish script executed successfully - [ ] File created at correct path (posts/) - [ ] URL accessible (HTTP 200) - [ ] Sitemap updated with new post - [ ] RSS feed includes new post - [ ] Canonical URL set correctly - [ ] Date published as intended - [ ] Author attribution present - [ ] Categories/tags applied correctly - [ ] Internal linking to/from post works

Content Quality Check: - [ ] Readability score ≥70 - [ ] Scannable format (bullets, headings, short paragraphs) - [ ] Clear thesis and structure - [ ] Practical examples included - [ ] Call-to-action compelling and clear - [ ] Links open to valid pages - [ ] No broken links - [ ] Images load correctly - [ ] No technical errors in rendering

Weekly Operating Rhythm:

Monday: Sprint Planning - Review backlog and select next topic - Assign agent roster for upcoming sprint - Review team_notes from previous sprints - Set sprint objectives and success metrics

Tuesday-Thursday: Sprint Execution - Execute one complete sprint per day - Move agents in strict sequence - Document findings in team_notes - Review quality gates between agents

Sprint Review Meeting (Friday, 30 minutes): - Review completed sprints - Discuss successes and challenges - Update team_notes with learnings - Identify process improvements - Plan next week's priorities - Adjust sprint parameters based on performance
